# 관리자 기능

## 관리자 페이지 목표

관리자는 어드민 계정으로만 접속해 유저가 생성한 게시물, 댓글, 대댓글을 관리한다.

## 관리자 접근 정책

- 관리자 페이지는 `/admin/login`에서 어드민 계정으로 로그인한 사용자만 접근할 수 있다.
- 일반 유저 계정은 관리자 로그인에 성공할 수 없거나, 로그인 후 `/admin` 접근이 차단되어야 한다.
- 프론트엔드는 `AdminRoute`에서 로그인 여부와 `role === "admin"`을 확인한다.
- 백엔드는 모든 `/api/admin/*` 요청에서 JWT와 관리자 권한을 검증한다.

## 관리자 게시물 관리

관리자는 다음 작업을 할 수 있다.

- 유저가 생성한 전체 게시물 목록 조회
- 게시물 상세 조회
- 운영 정책에 맞지 않는 게시물 수정
- 부적절한 게시물 삭제

게시물 목록에는 다음 정보를 표시한다.

| 항목 | 설명 |
| --- | --- |
| 제목 | 게시글 제목 |
| 작성자 | 게시글 작성자 |
| 작성일 | 게시글 생성일 |
| 수정일 | 마지막 수정일 |
| 조회수 | 게시글 조회수 |
| 상태 | 정상, 삭제됨, 숨김 등 |

## 관리자 댓글 및 대댓글 관리

관리자는 다음 작업을 할 수 있다.

- 전체 댓글 및 대댓글 목록 조회
- 특정 게시글의 댓글 및 대댓글 조회
- 부적절한 댓글 삭제
- 부적절한 대댓글 삭제

댓글 관리 목록에는 다음 정보를 표시한다.

| 항목 | 설명 |
| --- | --- |
| 원본 게시글 | 댓글이 속한 게시글 |
| 유형 | 댓글 또는 대댓글 |
| 작성자 | 댓글 작성자 |
| 내용 | 댓글 본문 |
| 좋아요 수 | 좋아요 집계 |
| 싫어요 수 | 싫어요 집계 |
| 작성일 | 댓글 생성일 |
| 상태 | 정상, 삭제됨 등 |

## 관리자 UI

- 관리자 페이지는 게시물 관리 탭과 댓글/대댓글 관리 탭으로 나눈다.
- 데스크톱에서는 표 형태로 표시한다.
- 모바일에서는 카드형 목록 또는 가로 스크롤을 사용한다.
- 삭제 작업에는 확인 모달을 적용한다.
- 관리자 페이지 장식은 메인 페이지보다 절제하되, 석기시대 콘셉트의 탭, 배지, 돌판 카드 정도는 유지한다.

## 보안 기준

- 관리자 API 권한 검증은 프론트엔드가 아닌 백엔드에서 반드시 수행한다.
- 관리자 계정은 일반 회원가입으로 만들 수 없게 한다.
- 관리자 삭제 기능은 로그를 남길 수 있도록 확장 가능하게 설계한다.
