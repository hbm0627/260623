# 댓글, 대댓글, 좋아요/싫어요 기능

## 댓글 기능

- 게시글 상세 페이지에서 댓글을 작성할 수 있다.
- 댓글 작성은 로그인한 유저만 가능하다.
- 댓글은 작성자 정보, 내용, 작성일, 수정일을 포함한다.
- 댓글 수정 및 삭제는 작성자 본인만 가능하다.
- 관리자는 관리자 페이지에서 댓글을 삭제할 수 있다.

## 대댓글 기능

- 댓글에는 답글 형태의 대댓글을 작성할 수 있다.
- 대댓글 작성은 로그인한 유저만 가능하다.
- 대댓글은 부모 댓글 ID를 가진다.
- 대댓글 수정 및 삭제는 작성자 본인만 가능하다.
- 관리자는 관리자 페이지에서 대댓글을 삭제할 수 있다.

## 좋아요/싫어요

- 댓글과 대댓글에는 좋아요 기능이 있다.
- 댓글과 대댓글에는 싫어요 기능이 있다.
- 좋아요와 싫어요는 로그인한 유저만 누를 수 있다.
- 한 유저는 하나의 댓글 또는 대댓글에 좋아요와 싫어요 중 하나만 선택할 수 있다.
- 이미 누른 반응을 다시 누르면 취소된다.
- 좋아요를 누른 상태에서 싫어요를 누르면 좋아요가 취소되고 싫어요로 변경된다.
- 싫어요를 누른 상태에서 좋아요를 누르면 싫어요가 취소되고 좋아요로 변경된다.

## UI 요구사항

- 댓글 입력창은 게시글 본문 아래에 배치한다.
- 대댓글 입력창은 댓글의 답글 버튼을 누르면 펼쳐진다.
- 좋아요/싫어요 버튼은 선택 상태가 명확히 구분되어야 한다.
- 댓글 작성 후 목록이 즉시 갱신되거나 갱신 중 상태를 표시한다.
- 삭제 시 확인 모달을 띄운다.

## 공룡 캐릭터 인터랙션

- 댓글이 없을 때 작은 공룡 캐릭터가 말풍선으로 첫 댓글 작성을 유도한다.
- 댓글 작성 성공 시 공룡이 짧게 점프하거나 손을 흔드는 애니메이션을 보여준다.
- 좋아요를 누르면 공룡 발자국 효과를 짧게 표시할 수 있다.
- 싫어요는 과하게 부정적으로 보이지 않도록 회색 돌 조각 아이콘이나 중립적 상태로 표현한다.

## 데이터 처리

- 댓글과 대댓글은 하나의 `Comment` 모델로 관리한다.
- 일반 댓글은 `parentId`가 `null`이다.
- 대댓글은 `parentId`에 부모 댓글 ID를 저장한다.
- 반응은 `CommentReaction` 모델로 분리해 사용자별 중복 반응을 방지한다.
