# 인증, 회원, 프로필 기능

## 회원가입

- 비회원은 이름, 이메일, 비밀번호로 회원가입할 수 있다.
- 이메일은 중복 가입을 허용하지 않는다.
- 비밀번호는 서버에서 bcrypt로 해싱해 저장한다.
- 기본 권한은 `user`이다.
- `admin` 권한은 일반 회원가입으로 생성하지 않는다.

## 로그인

- 일반 로그인은 `/login` 화면에서 처리한다.
- 이메일과 비밀번호가 일치하면 JWT Access Token을 발급한다.
- 로그인 후 토큰은 프론트엔드에서 보관하고 API 요청 시 Authorization 헤더에 포함한다.
- 로그인 실패 시 원인을 과도하게 노출하지 않고 일반적인 실패 메시지를 표시한다.

## 관리자 로그인

- 관리자 로그인은 `/admin/login`에서 분리해 처리한다.
- 어드민 계정만 관리자 로그인에 성공할 수 있다.
- 서버는 로그인 성공 전 사용자 `role` 값이 `admin`인지 확인한다.
- 일반 회원 계정은 관리자 페이지에 접근할 수 없다.

## 로그아웃

- 클라이언트에 저장된 인증 토큰을 제거한다.
- 로그아웃 후 보호된 페이지 접근 시 로그인 페이지로 이동한다.

## 프로필

- 로그인 사용자는 `/profile`에서 내 정보를 조회할 수 있다.
- 이름, 자기소개 등 프로필 정보를 수정할 수 있다.
- 내가 작성한 게시글 목록을 확인할 수 있다.
- 프로필 페이지에도 공룡 캐릭터를 가볍게 배치해 안내 메시지를 제공한다.

## 권한 정책

| 권한 | 가능 기능 |
| --- | --- |
| 비회원 | 메인, 회원가입, 로그인, 게시글 목록/상세 조회 |
| 일반 회원 | 프로필 관리, 게시글 작성, 본인 게시글 수정/삭제, 댓글/대댓글 작성, 댓글/대댓글 반응 |
| 관리자 | 관리자 로그인, 전체 게시물 관리, 댓글/대댓글 관리, 관리자 페이지 접근 |

## 프론트엔드 처리

- `ProtectedRoute`로 로그인 사용자 전용 페이지를 보호한다.
- `AdminRoute`로 관리자 전용 페이지를 보호한다.
- 프론트엔드 차단은 UX 보조 기능이며 최종 권한 검증은 백엔드에서 수행한다.

## 백엔드 처리

- `authMiddleware`에서 JWT를 검증한다.
- `adminMiddleware`에서 `role === "admin"`을 검증한다.
- 인증 실패는 `401 Unauthorized`, 권한 부족은 `403 Forbidden`으로 응답한다.
