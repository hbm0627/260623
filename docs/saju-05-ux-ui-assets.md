# UX/UI 및 이미지 제작 기준

## 참고 이미지

사용자가 제공하는 참고 이미지는 `screenshot` 폴더에 넣는다. 프론트엔드 UX/UI를 만들 때 이 폴더의 이미지를 우선 참고한다.

## 이미지 생성

서비스에 필요한 추가 이미지는 Codex 내부 이미지 생성 스킬을 활용해 만든다.

생성 대상 예시:

- 홈 화면 메인 비주얼
- 사주 결과 카드 배경
- 로그인/회원가입 화면 보조 이미지
- 기본 프로필 이미지
- 서비스 아이콘성 이미지
- 온보딩 또는 빈 상태 이미지

## 저장 위치

- 생성 이미지는 `media/generated`에 저장한다.
- 참고 이미지는 `media/references` 또는 `screenshot`에 저장한다.
- 로컬 테스트 업로드 이미지는 `media/uploads`에 저장한다.

## UX/UI 방향

- 사주 플랫폼의 분위기에 맞는 신뢰감, 몰입감, 접근성을 우선한다.
- 사용자가 생년월일/시간 정보를 입력하는 흐름은 단순하고 명확해야 한다.
- 결과 화면은 요약, 상세, 저장/공유 액션이 쉽게 보이도록 구성한다.
- 모바일 사용성을 우선 고려한다.
- 로그인 전후 상태를 명확히 구분한다.

## 주요 화면 후보

- 홈
- 회원가입
- 로그인
- 카카오톡 로그인 진입
- 사주 정보 입력
- AI 분석 로딩
- 분석 결과
- 분석 기록
- 마이페이지
- 프로필 수정

## 구현 전 확인할 자료

- `screenshot` 폴더의 참고 이미지
- `media/references`의 보조 참고 이미지
- 기존 문서의 기능 요구사항
- Supabase 인증/스토리지 정책
