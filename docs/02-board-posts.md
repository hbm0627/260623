# 게시판 및 게시글 기능

## 게시판 목표

개인 홈페이지 방문자와 회원이 글을 읽고, 로그인한 회원이 자신의 글을 작성하고 관리할 수 있는 게시판을 제공한다.

## 게시글 기능

- 게시글은 생성, 조회, 수정, 삭제가 가능해야 한다.
- 게시글 목록과 상세 페이지는 비회원도 조회할 수 있다.
- 게시글 작성은 로그인한 유저만 가능하다.
- 게시글 수정은 해당 게시글을 생성한 유저만 가능하다.
- 게시글 삭제는 해당 게시글을 생성한 유저만 가능하다.
- 관리자는 관리자 API를 통해 운영 목적상 게시글을 관리할 수 있다.

## 조회수

- 게시글 상세 페이지에 접근하면 조회수가 증가한다.
- 조회수는 게시글 목록과 상세 페이지에 표시할 수 있다.
- 같은 사용자의 반복 조회를 제한할지 여부는 구현 단계에서 결정한다.
- 기본 구현은 상세 조회 API 호출 시 `viewCount`를 1 증가시키는 방식으로 시작한다.

## 게시글 목록

게시글 목록에는 다음 정보를 표시한다.

| 항목 | 설명 |
| --- | --- |
| 제목 | 게시글 제목 |
| 작성자 | 작성자 이름 또는 닉네임 |
| 작성일 | 게시글 생성일 |
| 조회수 | 상세 페이지 조회 횟수 |
| 댓글 수 | 게시글에 달린 댓글과 대댓글 수 |

## 게시글 상세

게시글 상세 페이지에는 다음 요소가 필요하다.

- 제목
- 작성자
- 작성일 및 수정일
- 조회수
- 본문
- 수정/삭제 버튼: 작성자 본인에게만 표시
- 댓글 목록
- 대댓글 목록
- 좋아요/싫어요가 적용된 댓글 인터랙션

## UI 콘셉트

- 게시글 카드는 돌판 또는 동굴 벽화 느낌의 표면으로 표현할 수 있다.
- 최신 글은 작은 화석 배지 또는 공룡 발자국 아이콘으로 강조한다.
- 빈 게시판 상태에서는 귀여운 공룡이 “아직 아무 글도 없어요” 같은 말풍선을 보여준다.

## 예외 상황

- 삭제된 게시글은 목록에서 숨기거나 “삭제된 게시글입니다” 상태로 표시한다.
- 작성자 본인이 아닌 사용자가 수정/삭제를 요청하면 `403 Forbidden`을 반환한다.
- 로그인하지 않은 사용자가 글 작성을 시도하면 로그인 페이지로 이동한다.
