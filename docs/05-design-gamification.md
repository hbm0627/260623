# 디자인, 레퍼런스, 게이미피케이션

## 디자인 콘셉트

개성 있는 개인 홈페이지를 만들기 위해 귀여운 공룡 캐릭터와 석기시대 분위기를 결합한다. 사이트는 단순한 소개 페이지가 아니라 사용자가 공룡 캐릭터와 상호작용하는 작은 게임 같은 웹사이트로 만든다.

## 핵심 키워드

- 귀여운 공룡 캐릭터
- 석기시대
- 동굴 벽화
- 돌판 UI
- 나무 버튼
- 화석 배지
- 공룡 발자국
- 말풍선 대화
- 클릭 반응
- 스크롤 애니메이션
- 게이미피케이션

## 공룡 캐릭터 사용 방식

- 메인 페이지 히어로 영역에 대표 공룡 캐릭터를 배치한다.
- 공룡을 클릭하면 말풍선으로 짧은 대사를 보여준다.
- 페이지마다 공룡의 역할을 다르게 둔다.

| 페이지 | 공룡 역할 |
| --- | --- |
| 메인 | 방문자를 맞이하는 안내자 |
| 로그인 | “동굴 입장권을 확인할게요” 같은 로그인 안내 |
| 회원가입 | 새 부족원을 환영하는 캐릭터 |
| 프로필 | 사용자 정보를 알려주는 친구 |
| 게시판 | 글 작성을 유도하는 탐험가 |
| 게시글 상세 | 댓글 참여를 유도하는 진행자 |
| 관리자 | 돌판 기록을 정리하는 관리자 공룡 |

## 인터랙션 아이디어

- 공룡 클릭 시 말풍선 표시
- 말풍선 대사는 페이지 맥락에 따라 변경
- 스크롤 시 공룡이 옆에서 따라오거나 살짝 움직임
- 게시글 작성 성공 시 공룡 점프 애니메이션
- 댓글 작성 성공 시 공룡이 손 흔들기
- 좋아요 클릭 시 발자국 파티클 표시
- 메뉴 열기 시 돌문이 열리는 느낌의 전환
- 로딩 상태는 공룡이 돌을 굴리는 짧은 애니메이션으로 표현

## 로컬 샘플 이미지 참고

참고 이미지: `../test_image/01.png`

샘플 이미지에서 참고할 점:

- 넓은 여백을 둔 미니멀 히어로 화면
- 좌측 상단 로고, 우측 상단 메뉴
- 대형 세리프 타이포그래피
- 공룡 일러스트를 히어로 메시지와 겹쳐 배치하는 방식
- 흰색 또는 아이보리 배경과 검정 라인 아트의 대비

내 프로젝트에 적용할 때는 다음처럼 보완한다.

- 무서운 T-rex보다 귀여운 공룡 캐릭터로 변형한다.
- 텍스트와 공룡이 겹쳐도 가독성이 깨지지 않도록 레이어와 여백을 조정한다.
- 석기시대 색감인 바위 회색, 모래색, 나뭇잎 녹색, 점토색을 추가한다.
- 메인 페이지뿐 아니라 로그인, 게시판, 관리자 화면에도 캐릭터를 작게 반복 배치한다.

## 색상 방향

| 역할 | 색상 방향 |
| --- | --- |
| 기본 배경 | 아이보리, 따뜻한 연한 모래색 |
| 주요 텍스트 | 거의 검정에 가까운 차콜 |
| 포인트 | 나뭇잎 녹색, 점토 오렌지, 하늘색 |
| 카드/패널 | 돌판 회색, 연한 베이지 |
| 위험/삭제 | 붉은 점토색 |

## 이미지 생성 계획

Codex 내장 이미지 생성 기능을 활용해 다음 자산을 만든다.

| 자산 | 설명 | 저장 위치 예시 |
| --- | --- | --- |
| 로고 | 귀여운 공룡 또는 공룡 발자국 기반 로고 | `frontend/src/assets/logos/` |
| 프로필 이미지 | 운영자 분위기를 담은 캐릭터형 프로필 이미지 | `frontend/src/assets/images/` |
| 히어로 배경 | 석기시대 동굴 또는 초원 느낌의 배경 이미지 | `frontend/src/assets/images/` |
| 공룡 캐릭터 | 말풍선과 함께 사용할 대표 캐릭터 | `frontend/src/assets/images/` |
| 빈 상태 이미지 | 게시글/댓글이 없을 때 보여줄 이미지 | `frontend/src/assets/images/` |

## 이미지 생성 프롬프트 방향

- 귀여운 공룡 캐릭터
- 석기시대 느낌
- 친근하고 장난스러운 분위기
- 개인 홈페이지용
- 말풍선과 함께 사용하기 좋은 여백
- 모바일에서도 알아보기 쉬운 형태
- 워터마크와 읽기 어려운 텍스트 금지

## 모바일 UX

- 모바일 첫 화면에서도 공룡 캐릭터와 대표 문구가 함께 보여야 한다.
- 메뉴는 햄버거 버튼 또는 하단 내비게이션으로 구성한다.
- 공룡 말풍선은 화면을 가리지 않도록 하단 또는 캐릭터 주변에 제한된 크기로 표시한다.
- 게시글, 댓글, 관리자 목록은 카드형 UI로 전환한다.

## 스크린샷 활용

- 레퍼런스 페이지의 데스크톱/모바일 스크린샷을 저장한다.
- 색상, 타이포그래피, 여백, 캐릭터 배치, 메뉴 구조를 분석한다.
- 구현 후 내 페이지도 스크린샷으로 비교해 텍스트 겹침, 이미지 잘림, 모바일 가독성, 버튼 크기를 점검한다.
